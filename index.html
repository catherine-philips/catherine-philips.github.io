02
General-Purpose RAG Document Assistant
Problem: Users needed to query documents in plain language - without knowing which file contained what, or how to search across formats. Built end-to-end solo.

What I built: RAG system supporting PDF, Excel, CSV, PPT, and TXT. Hybrid retrieval (vector search + BM25 with RRF fusion) with cross-encoder reranking. Session-based multi-document querying with source citation so users always know where an answer came from.

Focus: Heavy emphasis on answer quality and UX - tested extensively for edge cases including multi-turn context drift, ambiguous queries, and cross-document conflicts.

03
Natural Language to SQL Assistant
Problem: Non-technical business users couldn't self-serve on data — every question went through analysts. The goal was to let them query metrics in plain English.

What I built: Two-module system - KPI queries and hypothesis testing - using prompt engineering with schema injection and few-shot examples. Tested across Gemini API and OpenAI GPT with the same structured test suite.

Testing: 62 structured scenarios covering aggregations, filters, joins, and nested logic. 77.5% end-to-end accuracy (48/62). Failure analysis drove each iteration of prompt design.
